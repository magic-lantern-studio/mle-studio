Digital Workprint (DWP)

Introduction

The Digital Workprint is the Magic Lantern title specification file format that facilitates title deployment to multiple platforms. The DWP specifies all the resources that accompany a title and is a repository for saved information about the development of a title. The DWP is designed to be an open standard and designates a LISP-like structure that can be extended by title developers and third-party developers.

The Digital Workprint is a simple object system with containership relations that has both an in-memory API and an ASCII file format. Items in the DWP are instances of C++ classes derived from a DWP base class (MLWorkprintItem). The base class defines the basic capabilities for all DWP items: identity, traversal, structure editing, and input/output. DWP items include scenes, sets, groups, actors, and properties as well as other resource and attribute specifications.

Atomic Data Types

A Digital Workprint supports the following basic data types:

int An integer value.
float A floating-point real value.
string A string value.
